# 设置界面重绘设计文档

## 1. 目标
- 让用户在 3 秒内完成两个核心操作：修改快捷键、切换开机自启。
- 消除无意义提示和装饰性控件，保证每个元素都有明确任务。
- 布局可扩展：新增设置项时，不需要手动重算全部坐标。

## 2. 信息架构
- 标题栏：页面名称 + 副标题 + 关闭按钮。
- 快捷键区（优先级最高）：
  - 快捷键预览卡片（展示当前/待保存组合）。
  - 状态提示文本（录制前/录制中/待保存）。
  - 主操作区按钮（开始录制 / 保存 / 取消）。
- 启动区（次优先级）：
  - 开机自动启动标题与说明。
  - 原生拨动开关（NSSwitch）放在右侧。

## 3. 布局规范
- 窗口尺寸：420 x 420。
- 内容边距：24。
- 区块间距：12。
- 标题栏保留高度：82。
- 纵向排布采用 top-down 线性流：每个组件通过 `place(height)` 获得矩形并推进游标。

### 3.1 纵向顺序（从上到下）
1. 快捷键预览卡片（88）
2. 状态提示（20）
3. 操作按钮行（42）
4. 开机自启卡片（64）

## 4. 交互规范
- 开始录制：进入录制态，显示“保存/取消”，隐藏“开始录制”。
- 保存：应用新快捷键，恢复默认态。
- 取消：放弃草稿，恢复默认态。
- 开机自启：采用原生开关；UI 状态直接由开关 `state` 驱动，不使用“伪按钮文案”表达状态。
- 开机状态读取失败时：禁用开关并回退到 off。

## 5. 实现约束
- `build_settings_form` 对外返回句柄不变：
  - `settings_hint_label`
  - `settings_preview_label`
  - `settings_record_button`
  - `settings_save_button`
  - `settings_cancel_button`
  - `settings_autostart_toggle`
- 保持现有 controller/state 接口，避免连带改动主流程。

## 6. 验收清单
- 设置页打开后无需滚动即可看到全部控件。
- “开始录制”在“开机自启”之上。
- 开机自启控件显示为系统拨动开关，而不是普通按钮。
- `cargo check` 通过，`cargo run` 可正常启动。
